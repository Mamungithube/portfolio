AJAXX Privacy Web

Personal data detection & automated removal from data broker sites

View on GitHub

Project Overview

AJAXX Privacy Web is a backend system built at SparkTech Agency that helps users take control of their personal data. In today's digital world, personal information — names, addresses, phone numbers, emails — gets collected and sold by hundreds of data broker companies without users' knowledge.

This system automatically scans major data broker websites, detects if a user's personal data is listed, and then submits opt-out and removal requests on their behalf — saving hours of manual work and protecting user privacy at scale.

Key Features

Automated Data Scanning

Scans 50+ data broker websites to check if a user's personal information (name, address, phone, email) is publicly listed. Uses structured scraping with rate-limiting to avoid detection.

Automated Opt-Out Requests

Automatically submits removal and opt-out requests to data brokers on behalf of the user. Handles form submissions, email verifications, and follow-up confirmations.

Background Task Processing

All scanning and removal jobs run asynchronously using Celery workers, so users get instant responses while heavy scraping tasks run in the background without blocking the API.

Real-time Status Tracking

Users can track the status of each removal request in real-time — whether it's pending, submitted, confirmed, or failed — through a clean REST API dashboard.

Secure JWT Authentication

Full JWT-based authentication with access/refresh token rotation. Users can only access and manage their own privacy scan data — no cross-user data exposure.

Email Notifications

Automated email alerts when data is found on a new broker site or when a removal request is confirmed, keeping users informed without needing to log in repeatedly.

How It Works

1

User Registration & Profile

User registers with name, email, phone, and address. The system stores this as the "search profile" that will be matched against broker databases.

2

Scan Initiation

User triggers a privacy scan via API. A Celery task is queued that dispatches sub-tasks for each data broker site to check in parallel.

3

Data Broker Scraping

Each sub-task hits a data broker's search page, looks for matching records, and reports findings back. Results are stored in PostgreSQL with full metadata.

4

Automated Removal Requests

For each positive match, the system submits a removal request to the broker — either via their opt-out form or direct email, depending on the broker's requirements.

5

Re-scan & Monitoring

Periodic re-scans are scheduled automatically to verify removal and catch any re-listing of data. Users are notified of any changes.
