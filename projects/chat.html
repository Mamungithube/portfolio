Real-time Chat System

WebSocket-powered instant messaging with JWT auth & message persistence

View on GitHub

Project Overview

This Real-time Chat System is a personal project built to master Django Channels and WebSocket communication — the key technologies behind any real-time application. The project implements a full-featured instant messaging backend that supports both private 1-on-1 conversations and group chat rooms.

Unlike a simple polling-based approach, this uses persistent WebSocket connections so messages are delivered instantly without any delay. Redis acts as the channel layer to enable message broadcasting across multiple server processes, making it production-scalable.

Key Features

Real-time Messaging via WebSocket

Messages are delivered instantly over persistent WebSocket connections. Zero polling — the moment a user sends a message, it appears on the recipient's screen in real-time without any page refresh.

Private & Group Chat Rooms

Supports 1-on-1 private conversations and multi-user group rooms. Users can create, join, and leave rooms. Each room has its own isolated WebSocket channel group.

Message Persistence

All messages are saved to PostgreSQL. New users joining a room can load full chat history via REST API. Message history is paginated and ordered chronologically.

Online Presence & Read Receipts

Users can see who is currently online in a room. Messages show sent/delivered/read status. Presence state is tracked in Redis and updated on WebSocket connect/disconnect events.

Typing Indicators

When a user is typing, a typing indicator event is broadcast to all room participants via WebSocket. Automatically clears after 3 seconds of inactivity.

JWT-based WebSocket Auth

WebSocket connections are authenticated using JWT tokens passed in the connection handshake. Unauthenticated connections are rejected immediately before the channel is established.

How It Works

1

WebSocket Handshake

Client connects to the WebSocket endpoint with JWT token. Django Channels authenticates the token and assigns the connection to the appropriate room's channel group in Redis.

2

Message Send

User sends a message as JSON over the WebSocket. The consumer receives it, saves it to PostgreSQL, then broadcasts it to all subscribers in the room's channel group.

3

Redis Channel Layer

Redis acts as the message broker between multiple Django Channels workers. This means the system scales horizontally — many server instances can share the same channel groups.

4

Delivery to Recipients

All connected clients in the room receive the message instantly via their own WebSocket connection. Read receipts are sent back as acknowledgment events.

5

Disconnect Handling

On disconnect, the consumer removes the user from the channel group and updates their online status in Redis. Other users see the presence indicator update instantly.
