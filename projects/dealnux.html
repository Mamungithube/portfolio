Dealnux

Smart e-commerce product comparison & best deal finder

View on GitHub

Project Overview

Dealnux is an e-commerce product comparison backend built at SparkTech Agency. The core idea is simple — when a user searches for a product, Dealnux fetches listings from multiple e-commerce platforms, compares prices, ratings, and availability, and surfaces the best deal.

It acts as a smart aggregator that saves users from manually visiting 10 different sites to find the best price. The backend handles product data ingestion, normalization, comparison logic, and delivers clean, ranked results via REST API.

Key Features

Multi-Platform Product Search

Aggregates product listings from multiple e-commerce sources for a single search query. Results are fetched in parallel for speed and normalized into a unified product schema.

Smart Price Comparison & Ranking

Products are ranked by a composite score factoring in price, seller rating, shipping cost, and availability. Users can sort by any attribute or use the AI-calculated "best value" score.

Price History Tracking

Product prices are tracked over time. Users can see if a "deal" is genuinely a discount or if the price was recently inflated. Historical data is stored and exposed via API.

Wishlist & Price Alerts

Users can save products to a wishlist and set price drop alerts. When a product's price falls below a threshold, the system automatically sends an email notification.

Advanced Filtering

Filter results by price range, rating, brand, seller, shipping speed, and availability. All filters are handled server-side for performance with paginated responses.

Redis Caching Layer

Popular search results are cached in Redis to reduce redundant external API calls. Cache invalidation is handled intelligently based on product data freshness.

How It Works

1

User Submits a Search Query

User hits the search API with a product name. The query is normalized and dispatched to all integrated data source adapters in parallel.

2

Data Fetching & Normalization

Each adapter fetches listings from its source (via API or scraping), extracts relevant fields, and maps them to a common product schema — name, price, rating, image, seller, link.

3

Comparison & Scoring

The comparison engine calculates a "value score" for each listing based on price relative to average, seller trust score, and product rating. Top results are ranked.

4

Results Returned & Cached

Ranked results are returned to the client as paginated JSON. The result set is cached in Redis for 30 minutes to serve repeated queries instantly.

5

Price Tracking Background Job

Celery beat runs periodic jobs to refresh prices on wishlisted products and triggers email alerts for price drops.
