Hotel Management

Room booking, reservation management & payment integration system

View on GitHub

Project Overview

This Hotel Management System is a personal project that simulates a real-world hotel booking backend. It handles the core operations of a hotel — room management, guest reservations, availability checking, and payment processing.

The main challenge was handling concurrent reservation conflicts (two guests booking the same room at the same time) and integrating Stripe for real payment flows. This project gave me practical experience with transactional database operations and third-party payment API integration.

Key Features

Room Management

Full CRUD for hotel rooms with attributes like room type, floor, capacity, amenities, price per night, and availability status. Admin can add, edit, or deactivate rooms.

Reservation System

Guests select check-in/check-out dates; the system checks real-time room availability and blocks the room for the booking window. Handles overlap detection to prevent double-bookings.

Stripe Payment Integration

Guests pay for reservations via Stripe. The system creates payment intents, handles webhook confirmation events, and marks reservations as confirmed only after successful payment.

Invoice Generation

Automatic invoice generation after confirmed booking with itemized breakdown — room price, taxes, service charges. Invoices are stored and retrievable via API.

Check-in / Check-out Management

Staff can mark guests as checked-in or checked-out. Room status updates automatically. Late check-out fees are calculated and can be added to the final invoice.

Cancellation & Refund

Guests can cancel reservations. Refund eligibility is calculated based on cancellation policy (free cancellation window, partial refund, no refund). Refunds triggered via Stripe API.

  • Designing state machines for reservation status (pending → confirmed → checked-in → checked-out)
